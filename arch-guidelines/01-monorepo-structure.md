# Monorepo Structure

Overview of the repository layout, workspace configuration, and tooling setup.

## Repository Layout

```
repo/
├─ apps/
│  ├─ api/                 # FastAPI application
│  └─ web/                 # Next.js application
│
├─ packages/
│  ├─ client-sdk/          # Generated TypeScript API client
│  ├─ ui/                  # Shared React UI components
│  ├─ config/              # Shared lint/ts/prettier configs
│  └─ types/               # Shared TypeScript types
│
├─ infra/
│  └─ terraform/
│     ├─ modules/          # Reusable infrastructure modules
│     ├─ environments/
│     │  ├─ dev/
│     │  ├─ staging/
│     │  └─ prod/
│     └─ bootstrap/        # Remote state setup
│
├─ docker/
│  ├─ api.Dockerfile
│  └─ web.Dockerfile
│
├─ scripts/                # Helper scripts
├─ .github/workflows/      # CI/CD workflows
├─ docker-compose.yml
├─ Makefile
├─ package.json            # Root workspace config
├─ pnpm-workspace.yaml
├─ turbo.json
└─ pyproject.toml          # Python tooling config (optional)
```

## Directory Purposes

### `apps/`

Deployable applications. Each app is self-contained with its own:
- Dependencies
- Build configuration
- Tests
- Dockerfile

**apps/api/**: FastAPI backend service
**apps/web/**: Next.js frontend application

### `packages/`

Shared code libraries. These are consumed by apps and other packages.

**packages/client-sdk/**: Auto-generated TypeScript client from FastAPI OpenAPI
- Generated by `pnpm generate:client`
- Imported as `@acme/client-sdk`

**packages/ui/**: Shared React UI components
- shadcn/ui or internal design system
- Imported as `@acme/ui`

**packages/config/**: Shared configuration
- ESLint configs
- TypeScript configs
- Prettier configs
- Tailwind configs

**packages/types/**: Shared TypeScript types
- Domain types
- API contract types
- Utility types

### `infra/`

Infrastructure-as-code using Terraform.

**infra/terraform/modules/**: Reusable, composable modules
- vpc/
- database/
- api_service/
- web_service/

**infra/terraform/environments/**: Environment-specific roots
- dev/
- staging/
- prod/

Each environment has its own state and variables.

## Workspace Configuration

### Root package.json

```json
{
  "name": "acme-platform",
  "private": true,
  "packageManager": "pnpm@9.0.0",
  "scripts": {
    "dev": "turbo run dev --parallel",
    "build": "turbo run build",
    "test": "turbo run test",
    "lint": "turbo run lint",
    "typecheck": "turbo run typecheck",
    "generate:client": "openapi-typescript http://localhost:8000/openapi.json -o packages/client-sdk/src/index.ts"
  },
  "devDependencies": {
    "turbo": "^2.0.0",
    "@acme/config": "workspace:*"
  }
}
```

### pnpm-workspace.yaml

```yaml
packages:
  - "apps/*"
  - "packages/*"
```

### turbo.json

```json
{
  "$schema": "https://turbo.build/schema.json",
  "globalDependencies": ["**/.env.*local"],
  "pipeline": {
    "build": {
      "dependsOn": ["^build"],
      "outputs": [".next/**", "!.next/cache/**", "dist/**"]
    },
    "dev": {
      "cache": false,
      "persistent": true
    },
    "test": {
      "dependsOn": ["build"]
    },
    "lint": {},
    "typecheck": {}
  }
}
```

## Key Decisions

### Why pnpm?

- Fast, disk-space efficient
- Built-in workspace support
- Strict dependency management
- Content-addressable store

### Why Turborepo?

- Task caching and parallelization
- Pipeline dependencies
- Remote caching support
- Incremental builds

### Workspace Dependencies

```json
{
  "dependencies": {
    "@acme/ui": "workspace:*",
    "@acme/client-sdk": "workspace:*",
    "@acme/config": "workspace:*"
  }
}
```

Use `workspace:*` to reference other packages in the monorepo. This:
- Links packages locally during development
- Replaces with actual versions on publish
- Enables atomic commits across packages

## Adding New Apps/Packages

### New App

1. Create directory: `apps/my-app/`
2. Initialize package.json:
   ```json
   {
     "name": "@acme/my-app",
     "private": true,
     "scripts": {
       "dev": "...",
       "build": "...",
       "test": "..."
     }
   }
   ```
3. Add to turbo.json pipeline if needed
4. Reference in root workspace

### New Package

1. Create directory: `packages/my-package/`
2. Initialize package.json with `"name": "@acme/my-package"`
3. Create `src/index.ts` entry point
4. Export public API
5. Add to turbo.json pipeline
6. Reference in consuming apps: `"@acme/my-package": "workspace:*"`

## Python in the Monorepo

Python apps (like `apps/api/`) are self-contained with:
- `pyproject.toml` for dependencies and tooling
- `uv` or `poetry` for package management
- `ruff` for linting/formatting
- `mypy` or `pyright` for type checking
- `pytest` for testing

The root may have a `pyproject.toml` for shared Python tooling configuration, but each app manages its own dependencies.
